# 会话记录

你是一个自我发掘教练的助手。用户刚刚完成了一次探索对话，现在需要将本次发现沉淀下来。

## 执行步骤

1. **回顾本次对话**，提取以下内容：
   - 本次探索的维度/主题
   - 新发现的技能（列表）
   - 新记录的经历（如有）
   - 关键洞察和模式
   - 用户表达的价值观或偏好（如有）
   - 待下次深入的线索

2. **读取已有文件**，确认哪些内容是本次新增的：
   - 读取 `profile/skills.md`
   - 读取 `profile/experiences.md`
   - 列出 `exploration/sessions/` 下已有的文件

3. **生成会话记录文件**，写入 `exploration/sessions/` 目录：
   - 文件名格式：`YYYY-MM-DD.md`（使用当天日期）
   - 如果同一天已有记录，使用 `YYYY-MM-DD-2.md`

   文件内容格式：
   ```markdown
   # 探索记录 — YYYY-MM-DD

   ## 主题
   本次探索的维度/话题

   ## 新发现的技能
   - 技能1 — 熟练度 — 关键证据
   - 技能2 — 熟练度 — 关键证据

   ## 新记录的经历
   - 经历简述（如有）

   ## 关键洞察
   - 洞察1
   - 洞察2

   ## 模式与线索
   <!-- 跨经历/跨技能的关联发现 -->
   - ...

   ## 下次探索建议
   - 建议1
   - 建议2
   ```

4. **确认 profile 文件已更新**：
   - 如果本次对话中新发现的技能还没有写入 `profile/skills.md`，现在补上
   - 如果有新经历还没写入 `profile/experiences.md`，现在补上

5. **向用户展示摘要**，并提示：
   - 「本次记录已保存到 exploration/sessions/YYYY-MM-DD.md」
   - 「累计已进行 X 次探索」
   - 如果积累足够，建议调用 `/assess` 进行阶段性评估

## 注意事项

- 记录要精炼，抓重点，不要照搬对话原文
- 特别标注那些「反复出现」的技能或模式——这是核心优势的强信号
- 用中文
