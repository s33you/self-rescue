# 会话记录

用户刚刚完成了一次探索对话，现在需要将本次发现沉淀下来。

所有数据文件位于 `.self-rescue/` 目录下。如果该目录不存在，先按 `/start` skill 的初始化逻辑创建。

## 执行步骤

1. **回顾本次对话**，提取：
   - 本次探索的维度/主题
   - 新发现的技能（列表）
   - 新记录的经历（如有）
   - 关键洞察和模式
   - 用户表达的价值观或偏好（如有）
   - 方案讨论进展（如果涉及 `/plan` 相关讨论）
   - 成长相关记录（如果涉及 `/grow` 讨论：新设目标、学习记录、回顾发现）
   - 待下次深入的线索

2. **读取已有文件**，确认哪些是本次新增：
   - `.self-rescue/profile/skills.md`
   - `.self-rescue/profile/experiences.md`
   - 列出 `.self-rescue/sessions/` 下已有文件

3. **生成会话记录**，写入 `.self-rescue/sessions/`：
   - 文件名：`YYYY-MM-DD.md`（同一天已有则用 `YYYY-MM-DD-2.md`）
   - 内容格式：
     ```markdown
     # 探索记录 — YYYY-MM-DD

     ## 主题
     本次探索的维度/话题

     ## 新发现的技能
     - 技能1 — 熟练度 — 关键证据

     ## 新记录的经历
     - 经历简述（如有）

     ## 关键洞察
     - 洞察1

     ## 模式与线索
     - 跨经历/跨技能的关联发现

     ## 方案进展（如有）
     - 涉及的方向
     - 推进到什么阶段
     - 关键决策

     ## 成长动态（如有）
     - 新设的学习目标
     - 记录的学习内容
     - 回顾发现

     ## 下次探索建议
     - 建议1
     ```

4. **确认 profile 文件已更新**：补上遗漏的技能或经历

5. **向用户展示摘要**：
   - 「本次记录已保存」
   - 「累计已进行 X 次探索」
   - 积累足够时建议 `/assess`

## 注意事项

- 记录要精炼，抓重点
- 特别标注反复出现的技能或模式
- 用中文
